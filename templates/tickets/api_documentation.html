API Tickets - Documentation

Retour
Introduction

L'API Tickets permet aux systèmes externes de vérifier et valider les tickets générés par GestSport.

Base URL: {{ request.scheme }}://{{ request.get_host }}/tickets/api/

Authentification: Non requise (API publique pour scanners)

Endpoints Disponibles
1. Vérifier un Ticket

URL:

GET /api/verify/{ticket_number}/

Exemple:

GET /tickets/api/verify/TKT-EC73506E/

Réponse (200 OK):

2. Valider un Ticket

URL:

POST /api/validate/{ticket_number}/

Corps:

{}

Réponse (200 OK):

{
    "success": true,
    "message": "Ticket validé avec succès",
    "ticket": {
        "id": 1,
        "ticket_number": "TKT-EC73506E",
        "used_at": "2026-02-15T14:05:00Z"
    },
    "reservation": {
        "id": 4,
        "user_name": "Marie Martin",
        "activity_title": "Football Senior",
        "terrain_name": "Terrain A"
    }
}
3. Informations Simplifiées

URL:

GET /api/info/{ticket_number}/

Usage:

Idéal pour les scanners qui affichent uniquement les informations essentielles

Réponse (200 OK):

{
    "ticket_number": "TKT-EC73506E",
    "status": "valid",
    "activity": "Football Senior",
    "date": "15/02/2026",
    "time": "14:00",
    "duration": "120 minutes",
    "terrain": "Terrain A",
    "participant": "Marie Martin"
}
Codes d'Erreur
Code Message Description
404 Ticket non trouvé Le numéro de ticket n'existe pas dans le système
400 Ticket déjà utilisé Le ticket a déjà été validé précédemment
400 Ticket non valide La réservation associée n'est pas confirmée
500 Erreur serveur Erreur interne du système
